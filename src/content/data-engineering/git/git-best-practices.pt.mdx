## Boas práticas e .gitignore

Você fecha este módulo com os hábitos que separam você do usuário casual.
Saber comandos é 30% do trabalho; os outros 70% são disciplina ao usá-los:
qual mensagem você escreve, quais arquivos ignora, quando ramifica.

### Mensagens de commit que valem a pena

Um commit é uma pequena unidade de mudança + uma mensagem. A **mensagem é
para humanos**: o `git log` que seu time vai ler dentro de 6 meses quando
algo quebrar.

**Regras mínimas:**

1. **Primeira linha ≤ 72 chars.** Imperativo, presente: "adiciona" não
   "adicionando" nem "adicionou".
2. **Linha em branco** entre subject e corpo (se tiver corpo).
3. **Corpo opcional** explicando o porquê.
4. **Prefixo semântico** (Conventional Commits): `feat`, `fix`, `docs`,
   `refactor`, `test`, `chore`, `perf`, `build`, `ci`.

Exemplo completo:

```
fix(pipeline): handle null amount in payments aggregation

Stripe webhook can send amount=null when a charge is voided
mid-processing. Previously this crashed the daily aggregation
with TypeError. Now we coalesce null to 0 and log a warning.

Refs: DATA-142
```

### Commits atômicos

Um commit deve representar **uma ideia coerente**. Se sua mensagem tem "e",
provavelmente são dois commits.

Ruim:

```
fix bug + add feature + update README
```

Bom (3 commits):

```
fix(api): retry on 5xx with exponential backoff
feat(metrics): add request latency histogram
docs: update setup instructions for new metrics
```

### .gitignore: o que nunca deve entrar

`.gitignore` é um arquivo (ou vários, aninhados) listando padrões que o Git
deve **ignorar completamente**.

Categorias para ignorar:

```
# Gerados / builds
dist/ build/ out/ *.pyc __pycache__/ node_modules/ .next/

# IDE / sistema
.vscode/ .idea/ .DS_Store Thumbs.db

# Logs e temp
*.log logs/ *.tmp .cache/

# Dados brutos / outputs
data/raw/ data/processed/ *.parquet *.csv *.xlsx

# Segredos
.env .env.local *.pem *.key secrets.json
```

Segredos são **o mais crítico**. Uma chave AWS commitada em repo público é
descoberta em menos de 5 minutos por bots. Se acontecer, considere a
chave comprometida e rote — apagar o commit não basta.

### Templates de .gitignore

```bash
curl https://raw.githubusercontent.com/github/gitignore/main/Python.gitignore > .gitignore
```

### Se você já commitou algo que não deveria

1. **Rote a credencial imediatamente** no provider.
2. Adicione `.env` ao `.gitignore`.
3. Remova do repo (sem deletar localmente):
   ```bash
   git rm --cached .env
   git commit -m "chore: remove .env from tracking"
   git push
   ```
4. Para apagar do **histórico inteiro**, use `git filter-repo` ou BFG.

### Outras boas práticas

- **Nomes de branches** com prefixo + kebab-case: `feature/null-handling`,
  `fix/api-timeout`.
- **Rebase em branches longas** sobre `main` para evitar conflitos
  acumulados.
- **`git pull --rebase` por padrão**:
  ```bash
  git config --global pull.rebase true
  ```
- **Sem WIP em main**. Use branch ou `git stash`.
- **Signed commits** se sua organização exige (GPG ou SSH).

### Erros comuns

- **Repos com `data/`, `node_modules/`, `__pycache__/` versionados**.
- **Mensagens "wip" ou "fix"**.
- **`force-push` em main**.
- **Branch de 6 meses sem rebase**.
- **Apagar `.git/`**.

## Exercício prático

<Exercise
  id="git-best-practices-exercise"
  title="Filtrar arquivos por .gitignore"
  description="Implemente `shouldIgnore(path, patterns)` retornando true se path bate com algum dos patterns simples."
  language="javascript"
  starterCode={`function shouldIgnore(path, patterns) {
  return false;
}

console.log(shouldIgnore('src/__pycache__/foo.pyc', ['__pycache__/']));
console.log(shouldIgnore('app.log', ['*.log']));
console.log(shouldIgnore('app.py', ['*.log']));`}
  solution={`function shouldIgnore(path, patterns) {
  for (const p of patterns) {
    if (p.endsWith('/')) {
      const dirName = p.slice(0, -1);
      if (path.split('/').includes(dirName)) return true;
    } else if (p.startsWith('*.')) {
      const ext = p.slice(1);
      if (path.endsWith(ext)) return true;
    } else {
      if (path === p || path.endsWith('/' + p)) return true;
    }
  }
  return false;
}
console.log(shouldIgnore('src/__pycache__/foo.pyc', ['__pycache__/']));
console.log(shouldIgnore('app.log', ['*.log']));
console.log(shouldIgnore('app.py', ['*.log']));`}
  hints={[
    'Para padrões com / no final, veja se algum segmento do path é o diretório.',
    'Para *.ext, basta path.endsWith(".ext").',
  ]}
  tests={[
    {
      description: 'diretório aninhado',
      assert: 'shouldIgnore("a/b/__pycache__/c.pyc", ["__pycache__/"]) === true',
    },
    {
      description: 'wildcard de extensão',
      assert: 'shouldIgnore("logs/app.log", ["*.log"]) === true',
    },
    {
      description: 'arquivo exato',
      assert: 'shouldIgnore(".env", [".env"]) === true',
    },
    {
      description: 'sem match',
      assert: 'shouldIgnore("src/main.py", ["*.log", "node_modules/"]) === false',
    },
  ]}
