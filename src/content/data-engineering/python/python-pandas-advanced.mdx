## Pandas avanzado: merge, pivot, groupby

La lección anterior cubrió los basics. Ahora viene la parte que de verdad
te hace productivo en transformaciones: **groupby** para agregar,
**merge** para joinear, y **pivot** para reorganizar.

### groupby: el equivalente a GROUP BY de SQL

```python
import pandas as pd
df = pd.read_parquet('payments.parquet')

# Una agregación
df.groupby('country')['amount'].sum()

# Varias agregaciones a la vez
df.groupby('country').agg(
    revenue=('amount', 'sum'),
    n_orders=('amount', 'count'),
    avg_ticket=('amount', 'mean'),
)

# Con varias columnas de grupo
df.groupby(['country', 'month']).agg({
    'amount': ['sum', 'mean'],
    'user_id': 'nunique',
})
```

Esa segunda forma con `('col', 'fn')` (named aggregation) es la
recomendada — produce DataFrames con columnas planas y nombres claros.

```python
# Reset del index para que el grupo sea una columna normal
result = (
    df.groupby('country')
      .agg(revenue=('amount', 'sum'))
      .reset_index()
)
```

### Filtros sobre grupos

```python
# Países con más de 100 órdenes (HAVING en SQL)
order_counts = df.groupby('country').size()
big_countries = order_counts[order_counts > 100].index
df_big = df[df['country'].isin(big_countries)]
```

O con `filter`:

```python
df.groupby('country').filter(lambda g: len(g) > 100)
```

### merge: JOINs en Pandas

```python
users = pd.DataFrame({'id':[1,2,3], 'name':['Ana','Beto','Carla']})
orders = pd.DataFrame({'user_id':[1,1,3], 'amount':[100,50,200]})

# INNER JOIN
users.merge(orders, left_on='id', right_on='user_id', how='inner')

# LEFT JOIN
users.merge(orders, left_on='id', right_on='user_id', how='left')

# Cuando se llaman igual:
users.rename(columns={'id':'user_id'}).merge(orders, on='user_id', how='left')
```

`how` acepta: `'inner'` (default), `'left'`, `'right'`, `'outer'`,
`'cross'`. Los mismos JOINs de SQL.

### concat: apilar DataFrames

```python
# Vertical (UNION ALL en SQL)
pd.concat([df1, df2], axis=0, ignore_index=True)

# Horizontal (raro, requiere mismo índice)
pd.concat([df1, df2], axis=1)
```

### pivot y pivot_table

```python
# pivot: long → wide. Falla si hay duplicados (key, columns)
df.pivot(index='date', columns='country', values='revenue')

# pivot_table: agrega duplicados con aggfunc
df.pivot_table(
    index='date',
    columns='country',
    values='amount',
    aggfunc='sum',
    fill_value=0,
)

# melt: wide → long (inverso de pivot)
wide.melt(id_vars=['date'], var_name='country', value_name='revenue')
```

### Window functions con transform

```python
# Suma total por usuario al lado de cada fila (sin colapsar)
df['user_total'] = df.groupby('user_id')['amount'].transform('sum')

# Rolling window (media móvil 7 días)
df.sort_values('date', inplace=True)
df['rolling_avg'] = df['amount'].rolling(window=7).mean()
```

`transform` mantiene la forma original; `agg` colapsa.

### Tips de performance

- **Lee con tipos correctos** (`dtype` argument o `pd.read_csv(..., dtype={'col':'int32'})`)
- **`category` para columnas con pocos valores únicos** (countries, status)
  ahorra muchísima RAM.
- **Vectoriza, no itera**. `apply` es mejor que un `for`, pero
  operaciones columnares directas (`df['x'] * 2`) son lo más rápido.
- **Para datasets grandes** (>10GB) considera **Polars** o **DuckDB** —
  Pandas es single-threaded y se queda corto.

### Errores comunes

- **`groupby` sin `as_index=False` o `reset_index()`** dejando la
  columna grupal como índice. Te tropieza al joinear o exportar.
- **`merge` que duplica filas** sin darte cuenta. Usa
  `validate='one_to_one'` o similar.
- **`pivot` con duplicados**: explota. Usa `pivot_table` con `aggfunc`.
- **Mutate en cadena cuando deberías reasignar**: `df.dropna()` no muta;
  hay que `df = df.dropna()` o `df.dropna(inplace=True)`.
