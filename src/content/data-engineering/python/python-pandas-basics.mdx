## Pandas fundamentos

**Pandas** es la librería de Python para manipulación de datos
tabulares. Si SQL es tu cuchillo en el warehouse, Pandas es tu cuchillo
fuera del warehouse: archivos CSV, APIs, transformaciones complejas en
memoria. Para un Data Engineer es herramienta diaria.

### Las dos estructuras

- **Series**: una columna unidimensional con índice.
- **DataFrame**: tabla 2D (filas × columnas), donde cada columna es una
  Series.

```python
import pandas as pd

# Crear DataFrame desde diccionario
df = pd.DataFrame({
    'user_id': [1, 2, 3, 4],
    'country': ['CO', 'MX', 'CO', 'AR'],
    'amount': [100, 50, 200, 75],
})
print(df)
```

### Cargar datos

```python
df = pd.read_csv('payments.csv')
df = pd.read_parquet('payments.parquet')
df = pd.read_json('events.json', lines=True)
df = pd.read_sql('SELECT * FROM payments', conn)
```

`read_parquet` es muchísimo más rápido que CSV para datasets grandes
(formato columnar, compresión).

### Inspección rápida

```python
df.head()           # primeras 5 filas
df.tail(10)         # últimas 10
df.info()           # tipos + memoria + nulls
df.describe()       # stats: count, mean, std, min, max, quartiles
df.shape            # (filas, columnas)
df.columns.tolist()
df.dtypes
```

`df.info()` es lo primero que corres con cualquier dataset nuevo. Te
dice si hay nulls inesperados, si los tipos son los correctos, y cuánta
RAM ocupa.

### Modificar columnas

```python
# Crear columna calculada
df['amount_with_tax'] = df['amount'] * 1.19

# Renombrar
df = df.rename(columns={'amount': 'gross_amount'})

# Castear tipo
df['user_id'] = df['user_id'].astype('int32')

# Aplicar función
df['country_code'] = df['country'].apply(lambda x: x.lower())
```

### Manejo de NaN

```python
df.isna().sum()                     # nulls por columna
df.dropna()                          # filas con cualquier NaN
df.dropna(subset=['amount'])         # solo si amount es NaN
df.fillna(0)                         # rellenar con 0
df['country'].fillna('unknown', inplace=False)
```

`inplace=True` modifica el DataFrame original sin devolver copia.
Generalmente se evita: prefiere reasignar.

### Diferencias clave con SQL

| SQL | Pandas |
|---|---|
| `WHERE` | `df.loc[mask]` |
| `SELECT col` | `df['col']` |
| `ORDER BY` | `df.sort_values('col')` |
| `GROUP BY` | `df.groupby('col')` (siguiente lección) |
| `JOIN` | `df.merge(df2)` (siguiente lección) |
| `LIMIT n` | `df.head(n)` |

### Errores comunes

- **`SettingWithCopyWarning`**. Cuando haces `df[df.x>0]['col'] = 1`,
  Pandas no sabe si actualiza el original o una copia. Usa `.loc[]`
  para ambos lados.
- **Operadores Python `and`/`or` en filtros de Series**. Crashea con
  "Truth value is ambiguous". Usa `&` y `|` con paréntesis.
- **`for row in df` para iterar**: lentísimo. Casi siempre hay una
  versión vectorizada (`apply`, `map`, operación columnar).
- **No verificar `df.dtypes`**. Una columna numérica leída como string
  rompe operaciones aritméticas silenciosamente.
