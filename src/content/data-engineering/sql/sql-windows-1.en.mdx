## Window functions I: ROW_NUMBER, RANK

**Window functions** are the difference between writing "basic" SQL and
"professional" SQL. They let you compute over a set of rows related to
the current row without collapsing the result like `GROUP BY` does. This
lesson covers ranking; the next covers LAG/LEAD/SUM OVER.

### The problem they solve

You want "top 3 best-selling products per category". Without window
functions, you need ugly correlated subqueries. With ROW_NUMBER, 4 lines:

```sql
WITH ranked AS (
  SELECT
    category,
    product_name,
    units_sold,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY units_sold DESC) AS rn
  FROM product_sales
)
SELECT category, product_name, units_sold
FROM ranked
WHERE rn <= 3;
```

### Anatomy

```sql
FUNCTION() OVER (
  PARTITION BY <col>     -- split into groups (optional)
  ORDER BY <col>         -- order within group
  -- ROWS BETWEEN ...    -- frame (next lesson)
)
```

- **PARTITION BY**: splits rows into groups. The function applies
  separately per group.
- **ORDER BY**: order within group.

Unlike `GROUP BY`, it does NOT collapse rows — original rows remain, with
the computed column added.

### ROW_NUMBER, RANK, DENSE_RANK

All three assign a number to each row in a group, but handle ties
differently:


Which to use?

- **ROW_NUMBER**: unique sequential id. "1st purchase", "5th impression".
- **RANK**: classic sports ranking (1, 2, 2, 4).
- **DENSE_RANK**: compact levels without gaps.

### Top N per group: the star pattern

```sql
WITH ranked AS (
  SELECT
    category,
    product_name,
    units_sold,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY units_sold DESC) AS rn
  FROM product_sales
)
SELECT category, product_name, units_sold
FROM ranked
WHERE rn <= 3;
```

### NTILE: deciles, quartiles

```sql
SELECT
  user_id,
  total_spent,
  NTILE(4) OVER (ORDER BY total_spent DESC) AS quartile
FROM user_revenue;
```

Quartile 1 = top 25% spenders. Useful for segmentation.

### vs GROUP BY

```sql
-- GROUP BY: collapses
SELECT category, AVG(price) FROM products GROUP BY category;
-- 3 rows

-- Window: preserves rows + adds column
SELECT
  category,
  product_name,
  price,
  AVG(price) OVER (PARTITION BY category) AS category_avg
FROM products;
-- N rows
```

### PARTITION BY without ORDER BY

```sql
SELECT
  user_id,
  amount,
  COUNT(*) OVER (PARTITION BY user_id) AS user_orders_count,
  SUM(amount) OVER (PARTITION BY user_id) AS user_total_spent
FROM payments;
```

Without ORDER BY inside OVER, computation applies to all rows in the
group: each row sees user total, not running.

### Common mistakes

- **`WHERE` filter on window** — can't do `WHERE ROW_NUMBER() OVER (...) <= 3`. Wrap in CTE.
- **Forgetting PARTITION BY** and getting global ranking.
- **`ROW_NUMBER` expecting tie consistency** — use RANK/DENSE_RANK.
- **`ORDER BY` in OVER misread** — it's the internal computation order,
  not the result order.

## Practical exercise

<Exercise
  id="sql-windows-1-exercise"
  title="ROW_NUMBER in JS"
  description="Implement `rowNumber(rows, partitionKey, orderKey)` returning rows with `rn` property = row_number within each partition ordered by orderKey desc."
  language="javascript"
  starterCode={`function rowNumber(rows, partitionKey, orderKey) {
  return rows;
}

const data = [
  {cat:'A',v:30},{cat:'A',v:10},{cat:'A',v:20},
  {cat:'B',v:5},{cat:'B',v:15},
];
console.log(rowNumber(data, 'cat', 'v'));`}
  solution={`function rowNumber(rows, partitionKey, orderKey) {
  const groups = {};
  for (const r of rows) {
    const k = r[partitionKey];
    (groups[k] ||= []).push(r);
  }
  const out = [];
  for (const arr of Object.values(groups)) {
    arr.sort((a, b) => b[orderKey] - a[orderKey]);
    arr.forEach((r, i) => out.push({ ...r, rn: i + 1 }));
  }
  return out;
}
const data = [
  {cat:'A',v:30},{cat:'A',v:10},{cat:'A',v:20},
  {cat:'B',v:5},{cat:'B',v:15},
];
console.log(rowNumber(data, 'cat', 'v'));`}
  hints={[
    'Group first, sort each group, then assign rn.',
    'Sort descending: (a,b) => b.v - a.v.',
  ]}
  tests={[
    {
      description: 'assigns rn correctly',
      assert: 'rowNumber([{c:"a",v:1},{c:"a",v:2}],"c","v").find(r=>r.v===2).rn === 1',
    },
    {
      description: 'resets rn per partition',
      assert: 'rowNumber([{c:"a",v:5},{c:"b",v:1}],"c","v").every(r=>r.rn===1)',
    },
    {
      description: 'preserves row count',
      assert: 'rowNumber([{c:"a",v:1},{c:"a",v:2},{c:"b",v:3}],"c","v").length === 3',
    },
  ]}
