## JOINs em profundidade

`JOIN` permite combinar linhas de duas ou mais tabelas baseado em uma
coluna relacionada. É o que faz o SQL ser SQL. Também é onde mais erros
bobos acontecem: uma condição mal escrita e seu relatório triplica as
vendas do mês.

### As 4 variantes que de fato você usa

<Diagram type="mermaid">
{`flowchart LR
  A[INNER JOIN<br/>só matches] --- B[LEFT JOIN<br/>tudo da esquerda<br/>+ matches]
  B --- C[RIGHT JOIN<br/>raro]
  C --- D[FULL OUTER JOIN<br/>tudo dos dois lados]`}
</Diagram>

Tabelas de exemplo:

```sql
-- users: id, name = 1/Ana, 2/Beto, 3/Carla
-- orders: id, user_id, amount = 10/1/100, 11/1/50, 12/4/80
```

### INNER JOIN: só matches

```sql
SELECT u.name, o.amount
FROM users u
INNER JOIN orders o ON u.id = o.user_id;
```

Resultado: Ana/100, Ana/50. Carla, Beto e order 12 não aparecem.
Interseção.

### LEFT JOIN: tudo da esquerda

```sql
SELECT u.name, o.amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;
```

Todos os users aparecem; quem não tem orders sai com NULL. JOIN mais
usado em analítica.

### RIGHT JOIN: pule

Quase sempre reescrito como LEFT JOIN com tabelas invertidas — mais
legível.

### FULL OUTER JOIN: tudo dos dois

Útil para detectar inconsistências.

### Erro clássico: explosão de linhas

3 orders × 4 items = 12 linhas, não 3. Para contar orders únicas use
`COUNT(DISTINCT o.id)`.

### Self JOIN

```sql
SELECT u1.name AS u1, u2.name AS u2, u1.country
FROM users u1
INNER JOIN users u2
  ON u1.country = u2.country
  AND u1.id < u2.id;
```

`u1.id < u2.id` evita duplicatas (Ana-Beto e Beto-Ana).

### USING vs ON

Prefira `ON` para clareza.

### Erros comuns

- **Esquecer condição do JOIN** — alguns motores tratam como CROSS JOIN.
- **JOIN com tabela duplicada** sem perceber.
- **LEFT JOIN seguido de WHERE em coluna da direita**: anula o LEFT.
- **Sem aliases**: queries com 4+ tabelas ficam ilegíveis.

## Exercício prático

<Exercise
  id="sql-joins-exercise"
  title="Simulador de INNER JOIN"
  description="Implemente `innerJoinSimple(left, right, leftKey, rightKey)`."
  language="javascript"
  starterCode={`function innerJoinSimple(left, right, leftKey, rightKey) {
  return [];
}

const users = [{id:1,name:'Ana'},{id:2,name:'Beto'}];
const orders = [{user_id:1,amount:100},{user_id:1,amount:50},{user_id:3,amount:80}];
console.log(innerJoinSimple(users, orders, 'id', 'user_id'));`}
  solution={`function innerJoinSimple(left, right, leftKey, rightKey) {
  const out = [];
  for (const l of left) {
    for (const r of right) {
      if (l[leftKey] === r[rightKey]) out.push({ ...l, ...r });
    }
  }
  return out;
}

const users = [{id:1,name:'Ana'},{id:2,name:'Beto'}];
const orders = [{user_id:1,amount:100},{user_id:1,amount:50},{user_id:3,amount:80}];
console.log(innerJoinSimple(users, orders, 'id', 'user_id'));`}
  hints={[
    'Loop sobre left, depois right; combine se chaves casarem.',
    'Use spread para mesclar objetos.',
  ]}
  tests={[
    {
      description: 'Match simples',
      assert: 'JSON.stringify(innerJoinSimple([{id:1,n:"a"}],[{uid:1,v:10}],"id","uid")) === \\\'[{"id":1,"n":"a","uid":1,"v":10}]\\\'',
    },
    {
      description: 'Sem matches',
      assert: 'innerJoinSimple([{id:1}],[{uid:2}],"id","uid").length === 0',
    },
    {
      description: 'Múltiplos matches multiplicam',
      assert: 'innerJoinSimple([{id:1}],[{uid:1},{uid:1}],"id","uid").length === 2',
    },
  ]}
/>
