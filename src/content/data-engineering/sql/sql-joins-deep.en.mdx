## JOINs in depth

`JOIN` lets you combine rows from two or more tables based on a related
column. It's what makes SQL SQL — without it you'd have a giant denormalized
table and many problems. It's also where most silly errors happen: a bad
ON condition and your monthly sales report triples.

### The 4 variants you actually use

<Diagram type="mermaid">
{`flowchart LR
  A[INNER JOIN<br/>matches only] --- B[LEFT JOIN<br/>everything from left<br/>+ matches]
  B --- C[RIGHT JOIN<br/>rare in practice]
  C --- D[FULL OUTER JOIN<br/>everything from both]`}
</Diagram>

Example tables:

```sql
-- users
id | name
1  | Ana
2  | Beto
3  | Carla

-- orders
id | user_id | amount
10 | 1       | 100
11 | 1       | 50
12 | 4       | 80
```

### INNER JOIN: matches only

```sql
SELECT u.name, o.amount
FROM users u
INNER JOIN orders o ON u.id = o.user_id;
```

```
name | amount
Ana  | 100
Ana  | 50
```

Carla doesn't show up (no orders). Beto neither. Order 12 (user_id=4)
either (no user 4). It's the intersection.

### LEFT JOIN: everything from the left

```sql
SELECT u.name, o.amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;
```

```
name  | amount
Ana   | 100
Ana   | 50
Beto  | NULL
Carla | NULL
```

All users appear. Most-used JOIN in analytics: "all users with their
orders if any".

### RIGHT JOIN: don't bother

```sql
SELECT u.name, o.amount
FROM users u
RIGHT JOIN orders o ON u.id = o.user_id;
```

Almost always rewritten as LEFT JOIN with tables flipped — more readable.

### FULL OUTER JOIN: everything from both

Brings non-matching rows from either side. Useful for inconsistency
detection.

### The classic error: row explosion

A customer has 3 orders; each order has 4 items. If you do:

```sql
SELECT u.id, COUNT(*) AS rows
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
LEFT JOIN order_items oi ON o.id = oi.order_id
GROUP BY u.id;
```

You'll see `12` rows for that user, not 3. The JOIN multiplied. To count
unique orders use `COUNT(DISTINCT o.id)`. To sum amounts without dups,
use aggregated subqueries or CTEs.

### Self JOIN

Same table joined to itself. Example: pairs of users from the same
country:

```sql
SELECT u1.name AS u1, u2.name AS u2, u1.country
FROM users u1
INNER JOIN users u2
  ON u1.country = u2.country
  AND u1.id < u2.id;
```

`u1.id < u2.id` avoids dupes (Ana-Beto and Beto-Ana).

### USING vs ON

```sql
INNER JOIN orders o ON u.id = o.id    -- explicit
INNER JOIN orders USING (id)          -- shorter
```

`USING` is elegant but confusing if column names match by accident.
Prefer `ON` for clarity.

### Common mistakes

- **Forgetting the JOIN condition** (writing `JOIN orders` without `ON`).
  Some engines treat as CROSS JOIN — total explosion.
- **JOINing a table with duplicates** unaware. Verify with
  `SELECT col, COUNT(*) FROM t GROUP BY col HAVING COUNT(*) > 1`.
- **LEFT JOIN followed by WHERE on right column**: nullifies the LEFT.
  Move filter to `ON`.
- **No aliases**: 4-table queries become unreadable.
- **Assuming INNER JOIN preserves order**. SQL doesn't guarantee order
  without `ORDER BY`.

## Practical exercise

<Exercise
  id="sql-joins-exercise"
  title="INNER JOIN simulator"
  description="Implement `innerJoinSimple(left, right, leftKey, rightKey)` taking two arrays of objects and returning the cartesian product filtered where left[leftKey] === right[rightKey]."
  language="javascript"
  starterCode={`function innerJoinSimple(left, right, leftKey, rightKey) {
  return [];
}

const users = [{id:1,name:'Ana'},{id:2,name:'Beto'}];
const orders = [{user_id:1,amount:100},{user_id:1,amount:50},{user_id:3,amount:80}];
console.log(innerJoinSimple(users, orders, 'id', 'user_id'));`}
  solution={`function innerJoinSimple(left, right, leftKey, rightKey) {
  const out = [];
  for (const l of left) {
    for (const r of right) {
      if (l[leftKey] === r[rightKey]) out.push({ ...l, ...r });
    }
  }
  return out;
}

const users = [{id:1,name:'Ana'},{id:2,name:'Beto'}];
const orders = [{user_id:1,amount:100},{user_id:1,amount:50},{user_id:3,amount:80}];
console.log(innerJoinSimple(users, orders, 'id', 'user_id'));`}
  hints={[
    'Loop over left, for each loop over right and combine if keys match.',
    'Use spread (...) to merge objects.',
  ]}

    },
    {
      description: 'No matches → empty',
      assert: 'innerJoinSimple([{id:1}],[{uid:2}],"id","uid").length === 0',
