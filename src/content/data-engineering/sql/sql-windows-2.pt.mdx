## Window functions II: LAG, LEAD, SUM OVER

As window functions da lição anterior dão ranking. As de hoje permitem
**olhar para outras filas** a partir da atual: anterior (`LAG`),
próxima (`LEAD`), ou todas as anteriores acumuladas (`SUM OVER`).
Indispensáveis para séries temporais.

### LAG e LEAD

  created_at,
  amount,
  LAG(amount, 1, 0) OVER (PARTITION BY user_id ORDER BY created_at) AS prev_amount,
  LEAD(amount, 1) OVER (PARTITION BY user_id ORDER BY created_at) AS next_amount,

### Soma acumulada

```sql
SELECT
  user_id, created_at, amount,
  SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at) AS running_total
FROM payments;
```

Com ORDER BY dentro do OVER, SUM acumula desde o início da partição até
a linha atual.

### Frame clauses

```sql
SUM(amount) OVER (
  PARTITION BY user_id
  ORDER BY created_at
  ROWS BETWEEN 6 PRECEDING AND CURRENT ROW   -- últimas 7 filas
)
```

Tipos: `ROWS BETWEEN N PRECEDING AND M FOLLOWING` (filas físicas);
`RANGE BETWEEN ...` (baseado em valores ORDER BY); `UNBOUNDED PRECEDING/
FOLLOWING`; `CURRENT ROW`.

### Média móvel de 7 dias

```sql
SELECT
  date,
  daily_revenue,
  AVG(daily_revenue) OVER (
    ORDER BY date
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS rolling_avg_7d
FROM daily_kpis;
```

### FIRST_VALUE e LAST_VALUE

<Callout type="warning" title="LAST_VALUE traiçoeiro">
  Frame padrão com ORDER BY termina em CURRENT ROW, então LAST_VALUE =
  linha atual. Para realmente a última, expanda com ROWS BETWEEN
  UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING.
</Callout>

```sql
LAST_VALUE(amount) OVER (
  PARTITION BY user_id
  ORDER BY created_at
  ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
) AS last_purchase
```

### Caso: primeira compra de cada user

```sql
WITH first_purchases AS (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY created_at) AS rn
  FROM payments
)
SELECT * FROM first_purchases WHERE rn = 1;
```

### Caso: retenção dia-sobre-dia

```sql
WITH daily AS (
  SELECT date_trunc('day', event_at) AS day,
         COUNT(DISTINCT user_id) AS dau
  FROM events GROUP BY 1
)
SELECT
  day, dau,
  LAG(dau) OVER (ORDER BY day) AS dau_yesterday,
  100.0 * (dau - LAG(dau) OVER (ORDER BY day)) / LAG(dau) OVER (ORDER BY day) AS pct_change
FROM daily ORDER BY day;
```

### Erros comuns

- **LAST_VALUE sem frame estendido**.
- **Esquecer ORDER BY em OVER quando precisa**.
- **Confundir ROWS e RANGE**.

## Exercício prático

<Exercise
  id="sql-windows-2-exercise"
  title="Soma acumulada"
  description="Implemente `runningTotal(rows, key)`."
  language="javascript"
  starterCode={`function runningTotal(rows, key) {
  return rows;
}

console.log(runningTotal([{v:10},{v:20},{v:5}], 'v'));`}
  solution={`function runningTotal(rows, key) {
  let acc = 0;
  return rows.map(r => {
    acc += r[key];
    return { ...r, total: acc };
  });
}
console.log(runningTotal([{v:10},{v:20},{v:5}], 'v'));`}
  hints={[
    'Mantenha um acumulador externo ao map.',
  ]}
  tests={[
    {
      description: 'soma acumulada',
      assert: 'runningTotal([{v:10},{v:20}],"v").map(r=>r.total).join(",") === "10,30"',
    },
    {
      description: 'array vazio',
      assert: 'runningTotal([],"v").length === 0',
    },
    {
      description: 'preserva campos',
      assert: 'runningTotal([{name:"a",v:5}],"v")[0].name === "a"',
    },
  ]}
