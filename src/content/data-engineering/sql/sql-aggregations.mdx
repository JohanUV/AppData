## Agregaciones y GROUP BY

Las agregaciones son la razón por la que SQL es tan poderoso para
analítica: convierten millones de filas crudas en métricas resumidas. En
lugar de mirar cada transacción individual, miras "ventas totales por
día y por país". `GROUP BY` es la cláusula que hace eso posible.

### Las funciones de agregación básicas

```sql
SELECT
  COUNT(*) AS rows_count,
  COUNT(DISTINCT user_id) AS unique_users,
  SUM(amount) AS total_revenue,
  AVG(amount) AS avg_ticket,
  MIN(amount) AS smallest_sale,
  MAX(amount) AS biggest_sale
FROM payments
WHERE status = 'paid';
```

Sin `GROUP BY`, todas estas funciones colapsan toda la tabla a una sola
fila. Es el "modo summary": una métrica de cada cosa.

`COUNT(*)` cuenta filas; `COUNT(col)` cuenta filas donde `col` no es
NULL; `COUNT(DISTINCT col)` cuenta valores únicos no nulos. Tres cosas
distintas — confúndelas y el reporte sale mal.

### GROUP BY: agrupar antes de agregar

```sql
SELECT
  country,
  COUNT(*) AS orders,
  SUM(amount) AS revenue
FROM payments
GROUP BY country;
```

Una fila por country. Cada fila resume todas las órdenes de ese país. Si
agregas más columnas al SELECT, deben aparecer también en GROUP BY (o
ser agregaciones):

```sql
-- ESTO FALLA en SQL estándar:
SELECT country, city, COUNT(*) FROM payments GROUP BY country;
-- Error: city no está en GROUP BY ni es agregación

-- BIEN:
SELECT country, city, COUNT(*) FROM payments GROUP BY country, city;
```

(MySQL antiguo era permisivo aquí, pero esa "comodidad" producía bugs
silenciosos. Postgres y SQL Server son estrictos.)

### El orden lógico revisado

Ahora con agregaciones:

<Diagram type="mermaid">
{`flowchart LR
  A[FROM] --> B[WHERE] --> C[GROUP BY] --> D[HAVING] --> E[SELECT] --> F[ORDER BY] --> G[LIMIT]`}
</Diagram>

`WHERE` filtra **antes** de agrupar; `HAVING` filtra **después**.
Confundirlos rompe queries:

```sql
-- ¿Países con más de 100 órdenes pagadas?

-- BIEN:
SELECT country, COUNT(*) AS n
FROM payments
WHERE status = 'paid'        -- filtra filas individuales
GROUP BY country
HAVING COUNT(*) > 100;        -- filtra grupos

-- MAL:
SELECT country, COUNT(*) AS n
FROM payments
WHERE COUNT(*) > 100          -- error: COUNT no existe aún en WHERE
GROUP BY country;
```

### Múltiples niveles de agregación

```sql
SELECT
  date_trunc('month', created_at) AS month,
  country,
  COUNT(*) AS orders,
  SUM(amount) AS revenue,
  AVG(amount) AS avg_ticket
FROM payments
WHERE status = 'paid'
GROUP BY 1, 2
ORDER BY 1 DESC, revenue DESC;
```

`GROUP BY 1, 2` agrupa por las dos primeras columnas del SELECT (mes y
country). `date_trunc` (Postgres) trunca un timestamp al inicio del
periodo — `'month'`, `'day'`, `'hour'`. Equivalentes:

- BigQuery: `DATE_TRUNC(created_at, MONTH)`
- Snowflake: `DATE_TRUNC('month', created_at)`
- MySQL: `DATE_FORMAT(created_at, '%Y-%m-01')`

### Métricas comunes en analítica

Patrones que vas a escribir mil veces:

```sql
-- DAU (daily active users)
SELECT date_trunc('day', event_at) AS day,
       COUNT(DISTINCT user_id) AS dau
FROM events GROUP BY 1;

-- Conversión de embudo
SELECT step, COUNT(DISTINCT user_id) AS users
FROM funnel_events GROUP BY step ORDER BY step;

-- Top N por categoría (sin window functions)
SELECT country, SUM(amount) AS revenue
FROM payments GROUP BY country
ORDER BY revenue DESC LIMIT 10;

-- AOV (average order value)
SELECT AVG(amount) AS aov FROM orders WHERE status = 'completed';
```

### Errores comunes

- **Olvidar columnas en GROUP BY** — error de SQL estándar.
- **`COUNT(col)` cuando querías `COUNT(*)`**: si `col` tiene NULLs,
  cuentas menos.
- **División entera** sin cast: `COUNT(x)/COUNT(*)` sale 0 en motores
  con división entera. Multiplica por `1.0` o castea.
- **Filtrar agregaciones en `WHERE`** — usa `HAVING`.
- **`AVG` sobre datasets con outliers**: una venta de $100k te tira el
  promedio. Considera `MEDIAN` (o `PERCENTILE_CONT`) para datos asimétricos.
- **`COUNT(DISTINCT)` sobre tablas enormes**: muy caro. Para
  aproximaciones rápidas, motores modernos tienen `APPROX_COUNT_DISTINCT`
  (HyperLogLog).

  solution={`function groupBy(rows, key, aggregator) {
  const buckets = {};
  for (const r of rows) {
    const k = r[key];
    if (!buckets[k]) buckets[k] = [];
    buckets[k].push(r);
  }
  return Object.entries(buckets).map(([k, group]) => ({
    [key]: k,
    value: aggregator(group),
  }));
}
const data = [
  {country:'CO', amount:10},
  {country:'CO', amount:20},
  {country:'MX', amount:15},
];
const result = groupBy(data, 'country', (rows) => rows.reduce((s,r)=>s+r.amount,0));
console.log(result);`}
