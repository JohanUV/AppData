## Window functions I: ROW_NUMBER, RANK

**Window functions** são a diferença entre escrever SQL "básico" e SQL
"profissional". Permitem cálculos em filas relacionadas à atual sem
colapsar como GROUP BY.

### O problema que resolvem

Você quer "top 3 produtos mais vendidos por categoria":

```sql
WITH ranked AS (
  SELECT
    category,
    product_name,
    units_sold,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY units_sold DESC) AS rn
  FROM product_sales
)
SELECT category, product_name, units_sold
FROM ranked
WHERE rn <= 3;
```

### Anatomia

```sql
FUNCAO() OVER (
  PARTITION BY <col>     -- divide em grupos (opcional)
  ORDER BY <col>         -- ordena dentro do grupo
)
```

Diferente de GROUP BY: NÃO colapsa linhas — todas as filas originais
permanecem, com a coluna calculada adicionada.

### Top N por grupo

```sql
WITH ranked AS (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY units_sold DESC) AS rn
  FROM product_sales
)
SELECT * FROM ranked WHERE rn <= 3;
```

### NTILE

```sql
SELECT user_id, total_spent,
  NTILE(4) OVER (ORDER BY total_spent DESC) AS quartile
FROM user_revenue;
```

### vs GROUP BY

```sql
-- GROUP BY: colapsa
SELECT category, AVG(price) FROM products GROUP BY category;

-- Window: preserva linhas + coluna
SELECT product_name, price,
  AVG(price) OVER (PARTITION BY category) AS category_avg
FROM products;
```

### PARTITION BY sem ORDER BY

```sql
SELECT user_id, amount,
  COUNT(*) OVER (PARTITION BY user_id) AS user_orders_count,
  SUM(amount) OVER (PARTITION BY user_id) AS user_total
FROM payments;
```

### Erros comuns

- **`WHERE` sobre window** — não pode. Envolva em CTE.
- **Esquecer PARTITION BY**.
- **`ROW_NUMBER` esperando consistência em empates** — use RANK.

## Exercício prático

<Exercise
  id="sql-windows-1-exercise"
  title="ROW_NUMBER em JS"
  description="Implemente `rowNumber(rows, partitionKey, orderKey)`."
  language="javascript"
  starterCode={`function rowNumber(rows, partitionKey, orderKey) {
  return rows;
}

const data = [
  {cat:'A',v:30},{cat:'A',v:10},{cat:'A',v:20},
  {cat:'B',v:5},{cat:'B',v:15},
];
console.log(rowNumber(data, 'cat', 'v'));`}
  solution={`function rowNumber(rows, partitionKey, orderKey) {
  const groups = {};
  for (const r of rows) {
    const k = r[partitionKey];
    (groups[k] ||= []).push(r);
  }
  const out = [];
  for (const arr of Object.values(groups)) {
    arr.sort((a, b) => b[orderKey] - a[orderKey]);
    arr.forEach((r, i) => out.push({ ...r, rn: i + 1 }));
  }
  return out;
}
const data = [
  {cat:'A',v:30},{cat:'A',v:10},{cat:'A',v:20},
  {cat:'B',v:5},{cat:'B',v:15},
];
console.log(rowNumber(data, 'cat', 'v'));`}
  hints={[
    'Agrupe primeiro, ordene cada grupo, depois atribua rn.',
  ]}
  tests={[
    {
      description: 'atribui rn corretamente',
      assert: 'rowNumber([{c:"a",v:1},{c:"a",v:2}],"c","v").find(r=>r.v===2).rn === 1',
    },
    {
      description: 'reinicia rn por partição',
      assert: 'rowNumber([{c:"a",v:5},{c:"b",v:1}],"c","v").every(r=>r.rn===1)',
    },
    {
      description: 'preserva contagem',
      assert: 'rowNumber([{c:"a",v:1},{c:"a",v:2},{c:"b",v:3}],"c","v").length === 3',
    },
  ]}
/>
