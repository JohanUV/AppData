## Normalización (1FN, 2FN, 3FN)

La **normalización** es el proceso de organizar columnas y tablas en
una base de datos para minimizar redundancia y prevenir anomalías
(actualizar lo mismo en varios lugares y olvidar uno). Es la base
del modelado **OLTP** (transaccional). En analytics OLAP (warehouse)
intencionalmente se desnormaliza — pero hay que entender la
normalización primero para saber qué se está rompiendo y por qué.

### El problema que resuelve

Imagina una tabla "todo en uno":

```
order_id | customer_name | customer_email | product | qty | price
1        | Ana           | ana@x.com      | Mouse   | 2   | 30
2        | Ana           | ana@x.com      | Teclado | 1   | 80
3        | Beto          | beto@y.com     | Mouse   | 5   | 30
```

Problemas:

- **Redundancia**: el email de Ana aparece en cada orden suya.
- **Anomalía de actualización**: si Ana cambia email, hay que tocar
  10 filas y arriesgar dejar una sin cambiar.
- **Anomalía de inserción**: ¿cómo agregas un cliente sin orden? No
  puedes (a menos que metas nulls feos).
- **Anomalía de eliminación**: borras la única orden de un cliente y
  pierdes su info.

### 1FN: Primera Forma Normal

Cada celda debe contener **un valor atómico** — sin listas, sin valores
multi-campo.

❌ No es 1FN:

```
order_id | products
1        | "Mouse, Teclado"
```

✓ 1FN:

```
order_id | product
1        | Mouse
1        | Teclado
```

### 2FN: Segunda Forma Normal

Está en 1FN **y** cada columna no-clave depende de la **clave completa**
(no solo parte de ella, en claves compuestas).

❌ Tabla `order_items` con clave `(order_id, product_id)`:

```
order_id | product_id | product_name | qty
```

`product_name` depende solo de `product_id`, no de la clave completa.
Está mal — se duplica.

✓ Separar:

```
order_items: (order_id, product_id, qty)
products: (product_id, product_name)
```

### 3FN: Tercera Forma Normal

Está en 2FN **y** ninguna columna no-clave depende **transitivamente**
de la clave (no hay dependencias entre no-claves).

❌ Tabla `customers`:

```
customer_id | name | city | country
```

Si `city` determina `country` (Bogotá → Colombia), entonces `country`
depende transitivamente vía `city`. Mover el país a una tabla `cities`:

```
customers: (customer_id, name, city_id)
cities: (city_id, name, country)
```

### El resultado: schema normalizado

Para nuestro ejemplo del inicio, después de aplicar 3FN:

Cada hecho aparece en un solo lugar; los IDs conectan tablas vía
foreign keys.

### Cuándo desnormalizar

OLTP (Postgres, MySQL) → 3FN típicamente. Optimizado para:

- Inserts/updates rápidos.
- Sin redundancia ni anomalías.
- Joins son baratos en motores OLTP con índices.

OLAP (BigQuery, Snowflake) → frecuentemente desnormaliza:

- Lecturas pesadas de columnas seleccionadas.
- Joins en datasets gigantes son caros (incluso en columnar).
- Storage es barato; copiar datos no es problema.

Por eso en warehouses ves "wide tables" con cientos de columnas o
schemas tipo star (próximas lecciones).

### Errores comunes

- **Normalizar a la fuerza en analytics**: 8 joins para una query
  simple. Mueve a wide table si las queries lo justifican.
- **Sobre-normalizar**: 4FN/5FN raramente añade valor en la práctica.
- **No normalizar nada**: tabla "Excel" gigante con anomalías.
- **Olvidar foreign keys** después de partir tablas. Sin FKs, los
  joins son caóticos.

  language="javascript"
  starterCode={`function violatesFirstNormalForm(rows) {
  return false;
}

console.log(violatesFirstNormalForm([{id:1, products:['a','b']}]));      // true
console.log(violatesFirstNormalForm([{id:1, products:'a, b, c'}]));     // true
console.log(violatesFirstNormalForm([{id:1, product:'mouse'}]));        // false`}
  solution={`function violatesFirstNormalForm(rows) {
  for (const r of rows) {
    for (const v of Object.values(r)) {
      if (Array.isArray(v)) return true;
      if (typeof v === 'string' && /,\\s*[a-z]/i.test(v)) return true;
    }
  }
  return false;
}
console.log(violatesFirstNormalForm([{id:1, products:['a','b']}]));
console.log(violatesFirstNormalForm([{id:1, products:'a, b, c'}]));
console.log(violatesFirstNormalForm([{id:1, product:'mouse'}]));`}
  hints={[
    'Recorre cada fila y cada valor.',
    'Array.isArray detecta arrays.',

  ]}

      assert: 'violatesFirstNormalForm([{p:["a","b"]}]) === true',
    },

    },
    {
