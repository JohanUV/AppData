## Tu primer DAG

Esta lección es práctica: vas a escribir, programar y correr tu
primer DAG en Airflow. Asumimos que ya tienes Airflow corriendo (pip
+ standalone o docker-compose).

### Setup local rápido

```bash
# Forma más rápida (modo standalone)
pip install apache-airflow
airflow standalone

# Te imprime la URL (http://localhost:8080) y un user admin con su password
```

`standalone` levanta scheduler + webserver + SQLite en una sola
terminal. **No para producción**, ideal para desarrollo.

Para algo más cercano a producción, usa
[official docker-compose](https://airflow.apache.org/docs/apache-airflow/stable/howto/docker-compose/).

### Estructura de un proyecto

```
my-airflow/
├── dags/
│   ├── daily_revenue.py
│   └── weekly_report.py
├── plugins/
└── airflow.cfg
```

`AIRFLOW_HOME` por default es `~/airflow`. Los DAGs viven en `dags/`
dentro de él. El scheduler escanea esa carpeta cada 30 segundos.

### El DAG paso a paso

```python
# dags/daily_revenue.py
from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.operators.bash import BashOperator

def extract_orders(**context):
    """Extrae órdenes del día anterior y guarda en /tmp."""
    execution_date = context['ds']    # YYYY-MM-DD del run actual
    print(f'Extrayendo órdenes para {execution_date}')
    # Aquí iría: requests.get(...) o psycopg2.execute(...)
    with open(f'/tmp/orders-{execution_date}.json', 'w') as f:
        f.write('[]')
    return execution_date              # esto va a XCom

def transform_data(**context):
    """Carga JSON, transforma con pandas, guarda como Parquet."""
    ti = context['ti']
    date = ti.xcom_pull(task_ids='extract_orders')
    print(f'Transformando para {date}')

default_args = {
    'owner': 'data',
    'depends_on_past': False,
    'retries': 2,
    'retry_delay': timedelta(minutes=5),
    'email_on_failure': False,
}

with DAG(
    dag_id='daily_revenue',
    default_args=default_args,
    description='Pipeline de revenue diario',
    schedule='@daily',
    start_date=datetime(2026, 1, 1),
    catchup=False,
    tags=['revenue', 'production'],
) as dag:

    extract = PythonOperator(
        task_id='extract_orders',
        python_callable=extract_orders,
    )

    transform = PythonOperator(
        task_id='transform_data',
        python_callable=transform_data,
    )

    load = BashOperator(
        task_id='load_to_warehouse',
        bash_command='echo "loading {{ ds }}" && sleep 2',
    )

    notify = BashOperator(
        task_id='notify_team',
        bash_command='echo "✓ revenue actualizado para {{ ds }}"',
    )

    extract >> transform >> load >> notify
```

### Probar el DAG

Antes de schedule, verifica que parsea:

```bash
# Verifica sintaxis y dependencias
python dags/daily_revenue.py

# Lista DAGs registrados
airflow dags list | grep daily_revenue

# Test sin tocar BD ni schedule
airflow dags test daily_revenue 2026-05-15

# Test de una sola tarea
airflow tasks test daily_revenue extract_orders 2026-05-15
```

`tasks test` corre la tarea ignorando dependencias y sin escribir al
metadata DB — perfecto para iterar.

### Trigger manual desde UI

1. Ve a http://localhost:8080
2. Login (admin / `password` mostrado por airflow standalone)
3. Encuentra `daily_revenue` en la lista
4. Toggle el switch para activarlo (sino sigue paused)
5. Botón "▷" para trigger inmediato

La UI te muestra el grafo, logs por tarea, gantt de tiempos, y el
historial.

### Qué pasa cuando falla

- Si una tarea falla y hay `retries: 2`, Airflow espera `retry_delay`
  y reintenta. Hasta 2 retries.
- Tras agotar retries, marca `failed`.
- Si `email_on_failure: True` y SMTP configurado, manda email.
- En la UI ves la tarea en rojo, con logs y stack trace completos.

### Errores comunes

- **DAG con `time.sleep` en el módulo (fuera de funciones)**: el
  scheduler relee el DAG cada 30s y se cuelga.
- **Imports lentos en el módulo**: idem. Mueve a la función `python_callable`.
- **Crear conexiones a BD en `python_callable` y olvidar cerrar**.
- **`catchup=True`** sin pensar — explosión de runs cuando levantas
  el scheduler.
- **Confiar en el orden de listas**: usa `>>` explícito, no asumas
  orden.

  solution={`function topoSort(deps) {
  const graph = {};
  const inDegree = {};
  for (const [from, to] of deps) {
    (graph[to] ||= []).push(from);
    inDegree[from] = (inDegree[from] || 0) + 1;
    inDegree[to] = inDegree[to] || 0;
  }
  const queue = Object.keys(inDegree).filter(n => inDegree[n] === 0);
  const result = [];
  while (queue.length) {
    const n = queue.shift();
    result.push(n);
    for (const m of graph[n] || []) {
      if (--inDegree[m] === 0) queue.push(m);
    }
  }
  return result;
}
console.log(topoSort([['transform','extract'],['load','transform']]));`}
